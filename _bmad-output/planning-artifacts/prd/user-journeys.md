# User Journeys

## Journey 1: 학습자 — 핵심 경험 (Happy Path)

**페르소나:** Seungbinmin, TOEFL 준비 중. 어휘는 외웠는데 지문 독해 시 계속 막히는 답답함을 느끼고 있다.

**Opening Scene:** 오전 9시, 망각 임박 알림을 받는다. 앱을 열면 세션 전 30초 워밍업 — 오늘의 단어들을 "아는 것 같다(→) / 불확실(←)"로 스와이프해 당일 퀴즈 순서가 자동 재배열된다.

**Rising Action:** 퀴즈 시작. Level 2 단어는 동의어 선택형, Level 4 단어는 문장 빈칸형으로 퀴즈 유형이 자동 결정된다. *deteriorate*에서 레이턴시 0.9초 — "이 단어, 이제 몸이 먼저 안다"는 느낌. AI가 생성한 오답(*diminish* vs *deteriorate*)이 실제 혼동 패턴이라 긴장감이 있다.

**Climax:** *precipitously*가 TOEFL 실제 지문 문단과 함께 등장한다. "The temperature dropped precipitously, causing..." — 논리적 맥락 속에서 단어를 처리한다. 틀렸다. "이 단어를 틀린 이유: 급격한 하강 뉘앙스를 *steeply*와 혼동" — 오답이 패널티가 아닌 진단 데이터로 제시된다.

**Resolution:** 세션 완료. "*precipitously* Level 3 → Level 4 진입, 레이턴시 1.4초 (전주 대비 -1.4초)." 숫자가 아니라 "이 단어는 이제 지문에서 멈추지 않는다"는 확신이 생긴다.

## Journey 2: 학습자 — 엣지 케이스 (공백 후 복귀)

**Opening Scene:** 2주 공백 후 앱을 열었다. 밀린 복습 단어 87개. 압도감에 앱을 닫고 싶다.

**Rising Action:** 14일 이상 공백 감지 → 복귀 세션 모드 자동 전환. Level 4~5 단어 중 망각 임박도 상위 10개만 추려 제안한다. "87개 다 하지 않아도 됩니다. 오늘은 10개만."

**Climax:** 세션 완료. 레이턴시가 전반적으로 늘었지만 시스템은 이를 "2주치 망각 데이터"로 처리한다. Level이 일부 하향 조정되고 복습 주기가 재설정된다.

**Resolution:** "복귀 완료. 87개 중 10개 처리. 내일 세션이 준비됩니다." 부담 없이 다시 궤도에 오른다.

## Journey 3: 관리자/운영 — 단어 DB 관리

**Opening Scene:** 새 TOEFL 모의고사에서 모르는 단어 8개를 발견했다.

**Rising Action:** 관리 페이지에서 단어 입력 → Python 스크래퍼 자동 실행 → TOEFL 코퍼스 예문·연어·논리 역할 수집 → 데이터 미리보기에서 확인 및 수동 수정.

**Climax:** 희귀 단어 — 스크래핑 실패. "데이터 없음 — 수동 입력 필요" 플래그 표시. 직접 예문을 작성하고 저장.

**Resolution:** 8개 단어 DB 등록, Level 1(미학습)로 초기화. 다음 세션부터 자동 포함된다.

| 공백 후 복귀 | 공백 감지 로직, 복귀 세션 모드, 망각 임박도 계산 |
| DB 관리 | 단어 추가 UI, 스크래퍼 트리거, 데이터 미리보기·수정, 수동 입력 폴백 |
